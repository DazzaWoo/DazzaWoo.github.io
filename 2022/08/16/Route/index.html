什麼是 Route?

Route 在 Rails 的架構下是整個網站的路徑對照表，就好比一個社區的管理員，當你到朋友家的社區拜訪時，此時管理員都會詢問你要找哪位，並引導你到正確的位置。而 Route 的路徑位置是符合 RESTful 設計。

什麼是 REST?

REST(Representational State Transfer 表現層狀態轉換)是一種全球資訊網軟體架構風格，目的是讓各式不同的軟體或程式在網路上能更便利的傳送資料。將每一個網址視為一個資源（resource），讓使用者透過URL存取。而每個資源也會對應到不同的HTTP動作。

在 routes.rb 輸入

Rails 會幫我們長出 8 條路徑並且對應到 Controller 7 個方法。
終端機輸入 rails routes 查看路徑。

7 個方法分別為：

  • index: 書籍列表
  • create: 新增書籍
  • new: 新增書籍頁面
  • edit: 編輯書籍頁面
  • show: 檢視單一書籍
  • update: 更新書籍
  • destroy: 刪除書籍

不需要這麼多方法時

可以加上 only 或 except 選出需要使用到的方法。

當 Resource 網址不夠用時

  1. 在 resource 裡用 get 的方式新增。
  • resources 為複數時，新增的路徑會 + 上 _id。
  • resources 為單數時，新增的路徑則沒有 id。
  1. 透過 member 或 collection 增加方法。
  • 用 member 增加方法路徑需要帶入 id 。
  • 用 member 增加方法路徑不需要 id 。
